Цели проекта

Создать централизованную систему для мгновенного доступа к актуальному расписанию, сократив время поиска информации с 5-10 минут до 10 секунд.

  • Охватить 90% студентов направления
  • Снизить количество опозданий на 40%
  • Автоматизировать процесс уведомлений

Системная архитектура

Микросервисная архитектура:

  • API Gateway - единая точка входа
  • User Service - управление пользователями
  • Schedule Service - работа с расписанием
  • Notification Service - система уведомлений
  • Analytics Service - сбор статистики

База данных

PostgreSQL схемы:

  • users - данные пользователей
  • groups - учебные группы
  • schedule - расписание занятий
  • notifications - история уведомлений
  • analytics - метрики использования

Производительность: 1000+ RPS, средняя задержка 15мс
